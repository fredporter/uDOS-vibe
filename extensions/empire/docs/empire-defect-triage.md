# Empire Defect Triage (Phase 1)

Scope
- Empire private extension only.
- Internal beta and alpha stabilization.

Severity model
- P0: release-blocking outage, data loss/corruption, security breach.
- P1: critical feature broken with no reasonable workaround.
- P2: degraded behavior with workaround; non-blocking for release.
- P3: minor defects, polish, and low-impact UX issues.

Ownership
- Every P0/P1 gets a single DRI and target fix date on creation.
- If no owner is assigned, the default owner is release lead.

Cadence
- Daily triage at local 10:00 for open defects.
- Immediate triage for any new P0.
