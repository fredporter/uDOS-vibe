# Installation

**Version:** v1.4.4+
**Last Updated:** 2026-02-22

Get uDOS-vibe up and running with Mistral Vibe CLI integration.

---

## Prerequisites

- **Python 3.10+** (3.12+ recommended)
- **uv** package manager (recommended) or pip
- **Git**
- **Terminal/Command Line** access

---

## Quick Install (Recommended)

Use the automated installer:

### macOS (Double-click)

1. Clone the repository
2. Double-click: `bin/install-udos-vibe.command`

### All Platforms (Terminal)

```bash
git clone --recurse-submodules https://github.com/mistralai/mistral-vibe.git uDOS-vibe
cd uDOS-vibe
./bin/install-udos-vibe.sh
```

The installer will:
- ✅ Detect your OS (macOS, Ubuntu, Alpine, Linux)
- ✅ Install uv package manager (if needed)
- ✅ Create .env with auto-configuration
- ✅ Install Vibe CLI
- ✅ Configure uDOS tools/skills
- ✅ Optional: Install Ollama + models
- ✅ Optional: Install Wizard server

---

## Manual Install

### 1. Clone Repository

```bash
git clone --recurse-submodules https://github.com/mistralai/mistral-vibe.git uDOS-vibe
cd uDOS-vibe
```

### 2. Install uv Package Manager

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### 3. Install Vibe CLI

```bash
uv tool install mistral-vibe
```

### 4. Configure Environment

```bash
cp .env.example .env
# Edit .env with your settings
```

### 5. Run Vibe

```bash
vibe  # or: uv run vibe
```

---

## Verify Installation

Once Vibe CLI is running:

```bash
vibe
# You should see Vibe CLI interface

# Try a command:
# User: "show me the map"
# → Routes to ucode_map tool

# Or use bash tool:
# User: "/bash ucode HELP"
# → Shows uDOS command reference
```

Shell commands:

```bash
# Direct command execution
ucode MAP
ucode GRID MAP
ucode BINDER LIST
```

## Troubleshooting

### Vibe Not Found

If `vibe` command not found:

```bash
uv tool install mistral-vibe
# or add to PATH:
export PATH="$HOME/.local/bin:$PATH"
```

### MCP Server Issues

Check MCP server status:

```bash
ucode STATUS
# or from Vibe:
# User: "/bash ucode STATUS"
```

### Python Version

Ensure Python 3.10+:

```bash
python3 --version
```
